## Graceful degradation acceptance framework

Add these acceptance criteria to new feature/bug issues unless explicitly irrelevant:

- [x] Unsupported, ambiguous, failed, or edge-case paths either complete the expected action or explain the next useful action in plain language.
- [x] Raw internal errors are logged, not shown as the primary user-facing message.
- [x] The UI distinguishes user-correctable conditions from system/service failures.
- [x] The failure path preserves user data and clipboard state.
- [x] The feature has at least one negative/manual QA case, not only a happy-path test.
- [x] Any fallback behavior is explicit in docs or issue-tracker comments, including what it does not cover
